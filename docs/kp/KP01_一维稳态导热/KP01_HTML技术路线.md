# KP01 · HTML 技术路线：一维稳态导热

> 本文档说明**如何设计**这个知识点专属的 HTML 页面——设计理念、交互隐喻、组件与实现逻辑，而非直接给出代码。核心目标：让页面形态与"一维稳态导热"这一物理本质**严丝合缝**，而不是套用通用模板。

---

## 1. 为什么这个知识点适合这样呈现？（设计立意）

一维稳态导热的三个可视化"抓手"极其明确：
1. **空间是一维的** → 页面主体就应该是一条**横向的墙体剖面**（从室外到库内），沿 x 方向铺开。
2. **温度沿厚度线性/分段线性下降** → 天然对应一条叠加在剖面上的**温度折线**。
3. **热阻串联 = 电阻串联** → 天然对应一个**"热阻梯子"/电路类比**面板。

因此本页的核心不是文字，而是一个**"可拖拽的分层墙体 + 实时温度折线 + 热阻电路类比"三联动**的交互模型。用户改变任何一层的厚度或材料，温度折线立即重绘、K 值实时刷新——把"保温层扛住大部分温差"这句话变成用户**亲手拖出来的现象**。

---

## 2. 核心交互隐喻（本页的"唯一记忆点"）

**"拖动保温层厚度，看温度折线怎么变陡、K 值怎么掉下来。"**

这是本页区别于其它 10 个页面的灵魂交互。别的页面绝不能有这个隐喻。

## 3. 页面布局与视觉结构

```
┌───────────────────────────────────────────────┐
│  标题：一维稳态导热 · 冷库外墙漏热                 │
├───────────────────────────────────────────────┤
│  [室外 30℃]  ┃外饰┃  混凝土  ┃  保温层  ┃衬板┃  [库内 -18℃] │   ← 剖面区（横向 x 轴）
│              温度折线 ╲___╲________________╲__          │   ← 折线叠加在剖面上
│  🌡 沿途温度色带（红→蓝渐变，映射温度）              │
├───────────────────────────────────────────────┤
│  热阻梯子（电路类比）: ─R_o─[R混]─[R保温]─[R衬]─R_i─ │   ← 每段长度∝热阻值
├───────────────────────────────────────────────┤
│  控制区：各层厚度滑杆 / 材料下拉 / 内外温度输入      │
│  指标卡：K=0.23 W/m²K  q=11.2 W/m²  内表面温=-16.6℃ │
└───────────────────────────────────────────────┘
```

**视觉语言**：横向 x 轴贯穿全页（呼应"一维"）；温度用红→蓝色带映射（左热右冷），让"冷"看得见；保温层用醒目的暖色边框，强调它是主角。

## 4. 关键交互组件与实现逻辑

### 4.1 分层墙体剖面（主视图）
- 用 **SVG** 绘制若干相邻矩形（每层一个 `<rect>`），宽度 ∝ 该层厚度 δ。
- 每层可**拖动右边界**改变厚度（拖拽手柄 → 更新 δ → 触发重算）。
- 每层填充色 = 材料标识色；顶部标注材料名与 λ 值。

### 4.2 温度折线（叠加层）
- 一条 `<polyline>`，顶点为各层界面处温度。
- 界面温度由热阻分压算出：$T_j = T_o - q\sum_{k\le j} R_k$。
- **关键呈现**：λ 小的层，折线斜率陡（ΔT 大）→ 用户一眼看到保温层"吃掉"最多温差。可在每段折线旁实时显示该层 ΔT 数值与占比条。

### 4.3 热阻梯子 / 电路类比（副视图）
- 一排串联的"电阻符号"，**每个电阻的绘制长度 ∝ 其热阻值 R=δ/λ**。
- 与上方剖面**同步高亮**：鼠标悬停某层，剖面该层与梯子对应电阻同时发光。
- 这是把"串联热阻"从公式变成可视比例的关键组件。

### 4.4 控制区
- 每层：厚度滑杆（也可拖剖面）、材料下拉（PU/XPS/EPS/混凝土……带预设 λ）。
- 全局：室外温度、库内温度、$h_o$、$h_i$ 输入。
- 一个"结露预警"指示灯：当内表面温度 < 库内露点时变红。

## 5. 页面实时计算的数学模型

页面需在每次输入变化时，纯前端即时求解（无需后端）：

```
R_o = 1/h_o
R_i = 1/h_i
R_layer[k] = δ[k] / λ[k]      // 逐层
R_total = R_o + ΣR_layer + R_i
K = 1 / R_total
q = K * (t_o - t_i)
// 界面温度（含表面）
T[0] = t_o - q*R_o            // 外表面
T[k] = T[k-1] - q*R_layer[k]  // 各层界面
t_内表面 = t_i + q*R_i
```

全部为代数式，`O(层数)` 复杂度，可在 `input`/`drag` 事件中同步计算并重绘，实现"零延迟"手感。

---

## 6. 可视化技术选型（为何这样选）

- **SVG（首选）**：墙体是少量矩形+折线+文字标注，SVG 的**可交互、可精确定位、可绑定事件**特性最契合；层界面拖拽、悬停高亮都容易实现。不用 Canvas，因为元素少且需要精细交互而非海量像素动画。
- **纯 JS + 响应式重算**：无需 D3，手写坐标映射即可；若想省事可用轻量 D3-scale 做温度↔颜色/坐标的比例尺。
- **温度色带**：用线性渐变 `linearGradient`，把温度映射到色相（30℃红 → −18℃蓝）。

## 7. 与主"数字冷库"页面的衔接

- 入口：用户在 3D/2D 冷库场景中点击**外墙**热点 → 浮窗显示简短说明"一维稳态导热……" → 浮窗底部"深入了解"按钮打开本页（模态层或独立路由 `/kp/01`）。
- 返回：本页保留"← 回到冷库"按钮，关闭时把用户拖出的参数（如保温厚度）通过 `postMessage`/URL 参数带回，可与其它页面（如 KP07 霜层、KP11 冷桥）共享"这堵墙"的设定，形成连贯教学线索。
- 视觉一致性：沿用主站配色与字体，但**本页独有的横向剖面 + 热阻梯子**是它的身份标识。

---

## 8. 技术栈建议

- 结构：原生 HTML + SVG；或 React + `svg`（若主站是 React，便于嵌入模态）。
- 状态：单一 `wallState`（层数组 + 环境参数），单向数据流：状态变 → 重算 → 重绘。
- 无需任何后端 / 数据库；参数可存 URL query 以便分享与回带。

---

## 9. 实现里程碑

1. 静态渲染分层剖面 + 固定温度折线（写死一组参数，验证坐标映射）。
2. 接入实时计算引擎（第 5 节），指标卡联动。
3. 加入滑杆/下拉控制，折线随参数变化。
4. 加入剖面边界拖拽改厚度（进阶手感）。
5. 加入热阻梯子与悬停同步高亮。
6. 加入结露预警与经济厚度提示曲线（可选进阶）。

---

## 10. 验收标准（"契合"与"完成"的判据）

- 拖动/加厚保温层，温度折线在该层明显变陡、K 与 q 明显下降——**现象由用户亲手拖出**。
- 悬停任一层，剖面与热阻梯子同步高亮，直观看到"哪层热阻大"。
- 计算结果与第 5 节公式、KP01 例题数值一致（K≈0.23、q≈11.2）。
- 页面形态（横向一维剖面 + 热阻电路）**只可能属于"一维稳态导热"**，换到别的知识点会明显不合适——这就是"不套模板"的达标线。
