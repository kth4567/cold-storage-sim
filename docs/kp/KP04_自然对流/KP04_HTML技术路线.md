# KP04 · HTML 技术路线：自然对流

> 说明"自然对流"专属 HTML 页面**如何设计**，非代码交付。

---

## 1. 为什么这个知识点适合这样呈现？

自然对流的核心是**看不见的空气因浮升力而自发流动，并在门面上形成边界层**。它的可视化必须把"流动"本身演出来，这决定了本页与所有"静态剖面/曲线"页面截然不同：
1. **有自发流动** → 页面主体应是一扇**竖直库门**，门面上有**随浮升力流动的空气粒子/流线**（冷面空气下沉、暖面空气上升），流动是本页的主角。
2. **边界层随温差变化** → 天然对应"温差滑杆 → 边界层变厚/变薄、粒子变快/变慢"。
3. **强弱由 Ra 决定** → 天然对应一个 **Ra→Nu→h** 的实时指示，并标出层流/湍流分界。

因此本页核心是**"门面自然对流粒子流场 + 边界层厚度 + Ra/Nu/h 仪表"**，是一个**"流动可视化"**页面。

---

## 2. 核心交互隐喻

**"调大门内外温差，看门面空气沉得更急、边界层更薄、h 蹿上去；温差归零，空气就静止不动了。"** 这是本页独有记忆点——**流动由温差自己'长'出来**。

## 3. 页面布局与视觉结构

```
┌───────────────────────────────────────────────┐
│  标题：自然对流 · 库门表面的浮升空气流            │
├───────────────────────┬───────────────────────┤
│   室内25℃              │  指标：ΔT | Gr | Ra    │
│   ┆  ┆  ┆ ← 空气粒子   │        Nu | h          │
│  ┃门┃ ↓↓↓ 冷面下沉流    │  流态灯：层流/湍流       │
│  ┃面┃ ▏边界层(渐变薄膜) │  Nu=C·Ra^n 曲线        │
│  ┃  ┃ ↓↓↓             │  (标当前工作点/分界)    │
│   库内-18℃             │                        │
├───────────────────────┴───────────────────────┤
│  控制：门表面温度 · 室内空气温度 · 门高L         │

└───────────────────────────────────────────────┘
```

**视觉语言**：门竖立居中，门面贴着一层**半透明边界层薄膜**（厚度可变），膜内密布**向下流动的粒子**（冷面）或向上（若表面比空气热）。粒子速度、边界层厚度都由物理量驱动，绝非装饰动画。

## 4. 关键交互组件与实现逻辑

### 4.1 门面自然对流流场（主视图，本页灵魂）
- **Canvas 粒子系统**：数百个空气粒子沿门面运动，速度 ∝ 由浮升力估算的特征速度（随 ΔT、L 变化）。
- 冷表面 → 粒子整体**下沉**；若切换成"表面比空气热"→ 粒子上升。方向本身传达物理。
- 温差=0 时粒子**静止**——强调"没有温差就没有自然对流"。

### 4.2 边界层可视化
- 门面外覆一层厚度 δ 的半透明渐变膜，$\delta$ 随 Ra 变化（Ra 大→薄→换热强）。
- 膜内叠加温度色带（表面色→主流色），呈现边界层内温度过渡。

### 4.3 Ra→Nu→h 仪表与关联式曲线
- **SVG** 画 $Nu=C\,Ra^n$ 分段曲线，标出**层流/湍流分界（Ra≈10⁹）**与当前工作点。
- 实时数字：Gr、Ra、Nu、h；流态指示灯（层流蓝/湍流橙）随 Ra 切换。

### 4.4 关/开门对比开关（承上启下）
- "关门"= 纯自然对流（本页主态）；"开门"= 门洞冷热空气剧烈对冲的强化流动，粒子暴增、漏热飙升——为 KP06 强制对流与渗透负荷埋下伏笔。
- 结露提示：门表面温度低于室内露点时，门面出现"凝露水珠"图元 + 提示"需防凝露加热"。

## 5. 页面实时计算模型

```
Tf = (Ts + T∞)/2 (K);  β = 1/Tf;  ΔT = |Ts - T∞|
Gr = g*β*ΔT*L^3 / ν^2
Ra = Gr * Pr
Nu = (Ra<1e9) ? 0.59*Ra^0.25 : 0.10*Ra^(1/3)
h  = Nu * λ / L
q  = h * ΔT
particleSpeed ∝ sqrt(g*β*ΔT*L)   // 特征浮升速度，用于粒子动画
δ_bl ∝ L / Nu                     // 边界层厚度示意（越换热越薄）
```

空气物性可随膜温微调或取常值；全部前端即时算。

## 6. 可视化技术选型（为何这样选）

- **Canvas 粒子系统**：自然对流的精髓是"流动"，需要成百上千粒子的连续动画，Canvas（或 WebGL 粒子）是唯一合理选择；SVG 撑不起这种量级动画。
- **SVG（Nu-Ra 曲线/仪表）**：曲线与工作点标注用 SVG 清晰。
- 与前三页（SVG 剖面 / Canvas 温度场）都不同：本页是**矢量场/粒子流**，技术选型直接呼应"对流=流动"这一物理本质。

## 7. 与主"数字冷库"页面的衔接

- 入口：主场景点击**库门**热点 → 浮窗"自然对流……" → "深入了解"进入本页。
- 联动：本页算出的门表面 $h$ 可回传 KP01，作为门体传热的表面换热热阻；"开门"态可跳转/联动 KP06 强制对流与渗透负荷教学。
- 身份标识：**门面浮升粒子流 + 边界层薄膜**是本页独有形态。

---

## 8. 技术栈建议

- Canvas 2D 粒子（或 `pixi.js`/WebGL 提升粒子数）+ SVG 曲线仪表。
- 状态：`{Ts, T∞, L}` → 派生所有物理量与动画参数，单向数据流。
- 纯前端，无后端。

---

## 9. 实现里程碑

1. 静态门 + Ra/Nu/h 计算与仪表（对齐例题 h≈3.35）。
2. 加粒子系统，速度/方向随 ΔT 变化，ΔT=0 静止。
3. 加边界层薄膜厚度随 Ra 变化。
4. 加 Nu-Ra 曲线与层流/湍流分界工作点。
5. 加关/开门对比与结露提示。

---

## 10. 验收标准

- 加大温差，粒子明显加速下沉、边界层变薄、h 上升；温差=0 时流动停止。
- Ra 跨过 10⁹ 时流态灯与关联式分支切换，工作点落在曲线正确分支。
- 数值与 KP04 例题一致（h≈3.35 W/m²·K）。
- 页面形态（**门面浮升粒子流**）**只可能属于"自然对流"**——与 KP06 强制对流的"风机吹动"形成对照，达成精准契合。
