# KP08 · HTML 技术路线：货物表面传热（Bi 判据）

> 说明"货物表面传热"专属 HTML 页面**如何设计**，非代码交付。

---

## 1. 为什么这个知识点适合这样呈现？

本知识点的灵魂是**"两段热阻的相对大小（Bi）决定瓶颈在内还是在外"**。最贴切的可视化不是流场、不是曲线，而是一个**"跷跷板/天平"**：一端是**表面对流热阻 1/h**，另一端是**内部导热热阻 L/λ**，Bi 就是它们的比值，跷跷板往哪边倾，瓶颈就在哪边。这个"比较两段热阻"的隐喻是别的知识点都没有的：
1. **两段串联热阻** → 主视觉是**货物剖面：外部冷风 →(对流膜)→ 表面 →(导热)→ 中心**的接力路径。
2. **Bi 是比值/判据** → 天然对应一个**热阻跷跷板/天平**，可视化 $1/h$ vs $L/\lambda$。
3. **Bi 决定归宿** → 天然对应一个**判据条**：Bi<0.1 亮"集中参数(KP10)"、Bi>0.1 亮"非稳态/中心滞后(KP09)"。

因此本页核心是**"降温接力路径 + Bi 热阻跷跷板 + 归宿判据分流"**——一个**"对比与判据"**页面，定位为 KP04/06（外）与 KP09/10（内）之间的"路由器"。

---

## 2. 核心交互隐喻

**"调 h、调货物尺寸/导热，看跷跷板往哪倒——倒向内部就中心滞后、倒向表面就整体同步；Bi 越过 0.1 时，页面自动把你分流到不同的处理方法。"** 这是本页独有记忆点。

---

## 3. 页面布局与视觉结构

```
┌───────────────────────────────────────────────┐
│  标题：货物表面传热 · 谁是降温瓶颈？(Bi)          │
├───────────────────────────────────────────────┤
│  冷风 -18℃ ⇒ [对流膜 1/h] │货物剖面│ 中心●        │
│              (第一棒)      (第二棒:导热 L/λ)      │
│  接力箭头 + 表面/中心温度实时读数                  │
├───────────────────────────────────────────────┤
│      热阻跷跷板:  1/h  ◁────▲────▷  L/λ            │
│      Bi = (L/λ)/(1/h) = h·L/λ  = 2.0             │
├───────────────────────────────────────────────┤
│  判据条: [Bi<0.1 集中参数→KP10] ●[Bi>0.1 非稳态→KP9]│
│  控制: h(风速) · 货物尺寸L · λ_goods · 包装层开关   │
└───────────────────────────────────────────────┘
```

**视觉语言**：中间一根**跷跷板**是全页焦点——左臂长度=1/h，右臂长度=L/λ，支点两侧不等则倾斜，倾向哪边一目了然。上方是"冷风→膜→货物→中心"的接力路径，下方是把用户"分流"到 KP09/KP10 的判据条。

---

## 4. 关键交互组件与实现逻辑

### 4.1 降温接力路径（上部）
- **SVG**：冷风区 → 半透明对流膜（厚度∝1/h 的相对感）→ 货物剖面（色带从表面到中心）→ 中心点。
- 两段用不同颜色的"接力箭头"，标注"第一棒：对流""第二棒：导热"。
- 表面温度、中心温度随参数与（可选）时间变化实时显示。

### 4.2 Bi 热阻跷跷板（主视图，本页灵魂）
- **SVG** 一根可倾斜的横杆：左端挂"1/h"块、右端挂"L/λ"块，块大小/力臂 ∝ 各自热阻。
- Bi = (L/λ)/(1/h)。Bi>1 右倾（内部瓶颈）、Bi<1 左倾（表面瓶颈）；实时显示 Bi 数值。
- 这是把抽象比值变成物理直觉的关键组件，别页不可复用。

### 4.3 归宿判据分流条（本页的"路由"角色）
- 一条带 0.1 阈值刻度的判据条，指针随 Bi 移动：
  - Bi<0.1 → 高亮"集中参数法"，提供"进入 KP10"按钮；
  - Bi>0.1 → 高亮"非稳态/中心滞后"，提供"进入 KP09"按钮。
- 让用户体验"先判 Bi、再选方法"的正确解题流程。

### 4.4 控制与包装开关
- h（风速，接 KP06）、货物尺寸 L、λ_goods 滑杆；"加包装层"开关在表面再叠一层导热热阻，跷跷板右倾、Bi 增大——直观演示"包装拖慢降温"。

---

## 5. 页面实时计算模型

```
R_conv = 1 / h                    // 表面对流热阻
R_cond = L_c / λ_goods            // 内部导热热阻(可 + 包装层 δ_pack/λ_pack)
Bi = R_cond / R_conv = h*L_c/λ_goods
tilt = f(log(Bi))                 // 跷跷板倾角，log 使左右对称
regime = (Bi < 0.1) ? "lumped(KP10)" : "transient(KP09)"
// 表面温度定性: Bi大→Ts≈T∞(风温); Bi小→Ts≈T_center
Ts ≈ T∞ + (T_center - T∞) * (R_conv/(R_conv+R_cond))
```

无需时间积分即可展示"瓶颈判据"；若要动态降温，可接 KP09 的非稳态解叠加。

---

## 6. 可视化技术选型（为何这样选）

- **SVG（跷跷板 + 接力路径 + 判据条）**：全是可精确定位、可动画倾斜、可标注的矢量元素，SVG 最合适，且跷跷板的"倾斜"用 SVG `transform` 极自然。
- 无需粒子或温度场——本页重点是**"比较两个数"而非"演流动"**，技术选型刻意"轻"，呼应其"判据/路由"定位。
- 与 KP06（粒子流场）区别明显：同讲对流，但本页讲的是**对流与导热谁大**，形态完全不同。

---

## 7. 与主"数字冷库"页面的衔接

- 入口：主场景点击**货物外表面**热点 → 浮窗"冷空气先与货物表面换热……" → "深入了解"进入本页。
- 路由角色：本页是"外部对流(KP04/06)"与"内部导热(KP09/10)"之间的**枢纽**，判据条上的按钮直接把用户送往正确的下一页。
- 身份标识：**Bi 热阻跷跷板 + 归宿判据分流**是本页独有形态。

---

## 8. 技术栈建议

- 纯 SVG + JS；跷跷板倾角、力臂长度由 Bi 派生。
- 状态 `{h, L, λ_goods, packaging}` → Bi、倾角、归宿、温度分配。
- 纯前端，无后端。

---

## 9. 实现里程碑

1. 静态接力路径 + 两段热阻块（写死参数），显示 Bi。
2. 加跷跷板倾斜随 Bi 变化。
3. 加判据条与阈值 0.1 指针、KP09/KP10 跳转按钮。
4. 加 h/L/λ 滑杆与包装开关联动（对齐例题 Bi=2.0 / 0.4）。
5. （进阶）叠加 KP09 非稳态解，动态显示表面/中心温度分离。

---

## 10. 验收标准

- 调参数时跷跷板正确倾斜、Bi 数值与倾向一致；加包装 Bi 增大右倾。
- 数值与 KP08 例题一致（R=0.1m→Bi=2.0；R=0.02m→Bi=0.4）。
- Bi 跨越 0.1 时判据条切换并能分流到 KP09/KP10。
- 页面形态（**Bi 跷跷板 + 判据路由**）**只可能属于"货物表面传热/Bi 判据"**——与 KP06 的流场页彻底区分，达成精准契合。
