# KP09 · HTML 技术路线：非稳态导热（中心温度滞后）

> 说明"非稳态导热"专属 HTML 页面**如何设计**，非代码交付。

---

## 1. 为什么这个知识点适合这样呈现？

本知识点的灵魂是**"时间"**：温度扰动由表及里、逐层滞后地渗透，**中心永远最后达标**。它与前面所有页面最大的不同——前面的页面大多在展示"稳态下的空间分布"或"某个比值"，而本页必须让用户**亲眼看到时间流逝中温度场的演化**。因此本页唯一正确的主视觉是**"会动的降温赛跑"**：

1. **双主角赛跑** → 表面温度线与中心温度线在同一张"温度—时间"图上**从同一起点出发、逐渐分离**，中心线明显"拖后腿"，最后才撞过达标线。这条"分离 + 滞后"是别的页面没有的记忆点。
2. **温度场随时间演化** → 货物剖面的颜色场不是静止的，而是**随播放进度不断由表及里变冷**，中间始终鼓着一个"最热核"。
3. **Bi 与 Fo 双旋钮** → Bi 改变"中心滞后有多严重"（曲线分得多开），Fo/时间是播放进度轴。

所以本页核心是**"可播放的降温赛跑 + 随时间演化的剖面温度场 + Heisler 查图"**——一个**以时间轴为骨架的动画页面**，是全套 11 页中**唯一强调"播放/时间流逝"**的一页。

---

## 2. 核心交互隐喻

**"按下播放，看表面线和中心线从同一起跑线出发；表面一骑绝尘先撞达标线，中心气喘吁吁最后才到——你能拖动 Bi 让中心'掉队得更狠'，也能读出中心撞线那一刻的时间，就是这批货的冷却时间。"** 这个"表面 vs 中心的赛跑与滞后"是本页独有记忆点。

**视觉语言**：全页由一根**时间轴（播放条）**统领，左右两个视图**共用同一个播放进度**——左边剖面在"变冷"，右边两条线在"赛跑"，形成"空间演化 ↔ 曲线分离"的双联动。中心那个**始终最热的"核"**是情感锚点：用户会盯着它什么时候变蓝。

## 4. 关键交互组件与实现逻辑

### 4.1 货物剖面温度场（左，Canvas）
- 一维/二维剖面（平板取一条厚度方向；圆柱/球取半剖面同心环）。
- 用 **一项近似 + 位置修正**（Fo>0.2）或预存的解，按当前 Fo 计算每一点的 $\theta^*(x,\text{Fo})$，映射到蓝—红色带。
- 中心用一个高亮的"**最热核**"标记，实时显示中心温度数字。
- 播放时颜色场平滑地"由表及里"变冷（requestAnimationFrame 驱动 Fo 递增）。

### 4.2 温度—时间赛跑图（右，SVG/Canvas）
- 两条曲线：**表面**（$x/L=1$，先降快）与**中心**（$x/L=0$，滞后）。可选加"平均温度"第三条。
- 横轴用 **Fo（无量纲时间）**，副轴换算成真实小时；纵轴用 $\theta^*$ 或真实温度双标。
- 一条水平**达标线**（如中心 −15 ℃）；中心线与它的交点**高亮闪烁**并标注"中心达标时间"。
- 播放头（竖直游标）随进度移动，与左侧剖面严格同步。

### 4.3 播放/时间控制（核心，全页独有）
- ▶播放 / ⏸暂停 / ↺重置；速度 x1/x2/x4；可拖动进度条**手动 scrub**（拖到哪，两个视图都跳到那一 Fo）。
- 这是全套页面里**唯一的时间轴**，务必做得顺滑、可逆、可逐帧。

### 4.4 Bi / 几何 旋钮
- 调 h（风速）或货物尺寸 L → 改变 Bi → 曲线"分离程度"实时变化（Bi 大，中心线掉得更狠、更靠后撞线）。
- 几何切换平板/圆柱/球 → 换用对应的 $C_1,\zeta_1$ 与位置函数，中心滞后规律随之变。
- 当 Bi 拖到 <0.1，页面给出提示气泡"**中心几乎不滞后 → 可用集中参数法(KP10)**"，形成向 KP10 的分流。

### 4.5 Heisler 查图小窗
- 一张简化的中心温度诺谟图（纵轴 $\theta^*_c$、横轴 Fo、参数 1/Bi），当前工作点用一个**发光圆点**落在对应曲线上，随播放沿曲线滑动。
- 让"查图法"和"动画"对应起来，帮学生把工程查图与物理过程建立联系。

---

## 5. 实时计算模型（前端可跑）

- **一项近似**（Fo>0.2）：$\theta^*_c=C_1e^{-\zeta_1^2\text{Fo}}$，位置修正 $\theta^*(x)=\theta^*_c\cdot f_{geom}(x/L,\text{Bi})$。$C_1,\zeta_1$ 按 Bi、几何**查内置小表并插值**。
- **早期（Fo<0.2）**：可切换到半无限大 erf 解（复用 KP03 的模型）以保证早段准确，形成 KP03↔KP09 的呼应。
- 时间反算：给定目标 $\theta^*_c$ → 解出 Fo → $t=\text{Fo}\,L_c^2/\alpha$，显示"冷却时间"。
- 全部为封闭表达式 + 小表插值，**浏览器实时**、无需服务器。

---

## 6. 可视化技术选型（为何这样选）

- **Canvas** 画温度色场与随时间的动画：像素级、逐帧刷新最流畅，适合"由表及里变冷"的连续演化。
- **SVG** 画赛跑曲线与达标线：矢量、易加游标/交点高亮/坐标轴，交互命中方便。
- **两者共享一个全局 Fo 状态**（单一数据源），保证左右视图帧帧同步——这是本页动画正确性的关键。

---

## 7. 与主站及相邻知识点的衔接

- 入口来自 **KP08** 的"Bi>0.1 → 非稳态"分流按钮，携带 Bi 值进入本页，曲线初始分离程度与 KP08 保持一致。
- 出口：Bi<0.1 提示跳 **KP10**；早期段说明跳 **KP03**；h 的来源链接回 **KP04/KP06**。
- 中心达标时间与放热量可回填到主站的**冷负荷/冻结时间**汇总。

---

## 8. 技术栈与里程碑

- **技术栈**：原生 Canvas（色场/动画）+ SVG（曲线）+ 少量 JS 状态机（播放控制）；无重型依赖。
- **里程碑**：
  1. 温度—时间双曲线（表面/中心分离）+ 播放条跑通；
  2. 左侧剖面色场随 Fo 演化、与右图同步；
  3. Bi/几何旋钮联动 + Heisler 小窗工作点滑动；
  4. Fo<0.2 半无限大切换 + 达标时间反算 + 分流跳转。

## 9. 验收标准

- 按播放，表面线与中心线**从同一点分离、中心明显滞后、最后撞达标线**，读数与一项近似计算一致（±5%）。
- 拖 Bi 变大 → 中心线掉队更狠、撞线更晚；拖到 <0.1 出现"跳 KP10"提示。
- 左右视图**帧帧同步**，可暂停、可 scrub、可重置。
- 页面**唯一**具备时间轴播放能力，与其余 10 页视觉/交互均不重复。
