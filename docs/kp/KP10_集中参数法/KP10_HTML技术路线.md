# KP10 · HTML 技术路线：集中参数法（内部温度均匀）

> 说明"集中参数法"专属 HTML 页面**如何设计**，非代码交付。

---

## 1. 为什么这个知识点适合这样呈现？

本知识点与 KP09 是一对"孪生对照"，但设计上必须**刻意做成 KP09 的反面**，让用户一眼看出区别：

- KP09 的灵魂是"**内部有温差、中心滞后**"——它的剖面是**彩色渐变**、曲线是**表面/中心两条分离线**。
- KP10 的灵魂是"**内部无温差、整块同温**"——它的剖面应当是**一整块均匀的纯色**（随时间整体变冷，但始终没有内部渐变），曲线是**唯一一条**平滑指数线。

这个"**均匀纯色块 vs 渐变温度场**""**单曲线 vs 双分离曲线**"的对照，本身就是最好的教学。此外，集中参数法的数学内核是 **RC 放电 / 水桶排水**，所以本页最贴切的隐喻是一支**"排空的温度计 / 漏水的桶"**，配一条**单指数衰减曲线**，再加一道**Bi<0.1 的准入闸门**（守住方法的适用边界）。

因此本页核心是**"均匀纯色块 + 单指数衰减曲线 + Bi<0.1 准入闸门 + 时间常数 τ 标尺"**——一个**极简、克制、与 KP09 强对照**的页面。

---

## 2. 核心交互隐喻

**"这块小货物是一支整体均匀的温度计（或一只漏水的桶）：它没有内外之分，整根水银柱/整桶水一起下降，节奏是纯粹的指数衰减，快慢由一个数 τ 决定。但进门要先验票——Bi 必须 <0.1，否则闸门变红、把你请去 KP09。"** 这个"均匀排空 + 验票闸门"是本页独有记忆点。

## 4. 关键交互组件与实现逻辑

### 4.1 Bi 准入闸门（本页独有"守门人"）
- 一个显眼的门/票闸控件，实时显示 Bi=hL_c/λ 与判定：**<0.1 绿灯"通过"、≥0.1 红灯"不适用"**。
- 红灯时闸门"关闭"、主视图变灰，弹出气泡"**内部温差不可忽略 → 请用非稳态导热(KP09)**"并给跳转按钮。
- 这一组件把"方法适用边界"变成可玩的交互，是本页教学重点。

### 4.2 均匀纯色货物块（左下，与 KP09 对照）
- 一个简单几何体（球/立方/薄片），**整体填充单一颜色**，随时间由暖到冷**整块变化**，**刻意不做任何内部渐变**。
- 旁边并排放一个"KP09 对照缩略图"（带渐变的剖面），一句话点明差别："这里整块同温，那里中心滞后。"

### 4.3 温度计 / 漏水桶（隐喻锚点）
- 一支温度计整根水银柱下降，或一只桶整体水位下降——强调"没有分层、整体排空"。液面高度 = $\theta^*(t)$。

### 4.4 单指数衰减曲线（右，SVG）
- 唯一一条 $\theta^*=e^{-t/\tau}$ 曲线；横轴标 τ、3τ、5τ 网格线，纵轴标 1、0.368、0.05 参考线。
- 曲线与 0.368 线的交点高亮为"τ"，与 0.05 线交点标"≈达标"。
- 拖动 h/尺寸改变 τ，曲线**横向拉伸/压缩**（τ 大变缓、τ 小变陡），实时联动。

### 4.5 τ 标尺 + 参数旋钮
- 旋钮：h（风速）、$L_c=V/A$（切小/变薄）、λ、ρc。每调一个，实时重算 $\tau=\rho V c/(hA)$ 并在"τ 标尺"上滑动。
- 直观展示强化手段：↑h 或 ↓L_c 都让 τ 变小、降温变快。

### 4.6 放热量读数
- 实时显示 $Q(t)=\rho Vc(T_i-T_\infty)(1-e^{-t/\tau})$，接入主站冷负荷汇总。

---

## 5. 实时计算模型（前端可跑）

- 判据：$\text{Bi}=hL_c/\lambda$，$L_c=V/A$（按几何自动取 r/3、r/2、半厚）。
- 核心：$\tau=\rho Vc/(hA)$；$\theta^*(t)=e^{-t/\tau}$；达标时间 $t=-\tau\ln\theta^*_{target}$。
- 放热量：$Q(t)=\rho Vc\,\Delta T_i(1-e^{-t/\tau})$。
- 全为初等封闭式，**浏览器瞬时**求解，可做实时滑杆联动。

---

## 6. 可视化技术选型（为何这样选）

- 本页刻意**极简**：主体用 **SVG**（曲线、闸门、温度计、色块）即可，不需要 Canvas 粒子/热力场——因为"均匀"本身意味着**不需要复杂的空间渲染**。这种"技术上的克制"正是对"内部无温差"的呼应。
- 单指数曲线用 SVG path，随 τ 做仿射横向缩放，交互轻快。
- 与 KP09 的"Canvas 富色场 + 双曲线"形成技术选型层面的对照，强化两页的差异记忆。

---

## 7. 与主站及相邻知识点的衔接

- 入口通常来自 **KP08**（Bi<0.1 分流）或 **KP09**（把 Bi 拖到 <0.1 时提示"可用更简单的集中参数法"）。
- 出口：闸门红灯 → 跳 **KP09**；h 来源 → **KP06/KP04**；τ=RC 的电路类比 → 呼应 **KP01**。
- $Q(t)$ 回填主站**冷负荷**汇总。

---

## 8. 技术栈与里程碑

- **技术栈**：纯 SVG + 少量 JS；无重依赖，体量最小的一页。
- **里程碑**：
  1. 单指数曲线 + τ/3τ/5τ 刻度 + 参数联动；
  2. 均匀纯色块整体变冷 + 温度计/水桶隐喻；
  3. Bi 准入闸门（绿/红 + 跳 KP09）；
  4. τ 标尺、放热量、与 KP09 对照缩略图。

## 9. 验收标准

- 主色块**始终无内部渐变**、整块随时间变冷——与 KP09 的渐变剖面形成肉眼可辨的对照。
- 曲线为**唯一一条**指数线，τ、3τ、5τ 刻度正确，拖 h/尺寸时横向缩放与 $\tau=\rho Vc/hA$ 一致。
- Bi≥0.1 时闸门变红、主视图置灰并给出跳 KP09 入口。
- 页面观感**极简克制**，与其余 10 页（尤其 KP09）在视觉与技术选型上均不重复。
