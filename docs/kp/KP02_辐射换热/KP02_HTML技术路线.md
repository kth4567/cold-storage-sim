# KP02 · HTML 技术路线：辐射换热

> 说明**如何设计**"辐射换热"专属 HTML 页面的技术路线与交互立意，非代码交付。

---

## 1. 为什么这个知识点适合这样呈现？

辐射换热的可视化"抓手"与导热完全不同：
1. **它来自天上（太阳/天空）** → 页面主视觉应是**一片天空 + 一轮可移动的太阳 + 一块冷库屋面**，用"日轨/日照角度"承载太阳辐照度的变化。
2. **屋面颜色决定吸收多少** → 天然对应一个**屋面颜色/反射率切换器**，切换即见屋面温度与漏热跳变。
3. **正比于 T⁴** → 天然对应一条**四次方曲线**，让用户直观感受"温度稍升、辐射剧增"。

因此本页核心是**"太阳日轨 + 屋面刷白开关 + T⁴ 曲线"**的场景化模拟：拖动太阳走过一天，切换屋面白/黑，实时看综合温度 $t_{sa}$ 与漏热如何随之起伏。它是一个**"环境驱动"**的页面，与 KP01 那种"结构剖面"页面形成鲜明对照。

---

## 2. 核心交互隐喻

**"把屋面从黑刷成白，看太阳灌进来的热瞬间少一半。"** 外加"拖动太阳走一天，看漏热曲线随日照起落"。这是本页独有的记忆点。

---

## 3. 页面布局与视觉结构

```
┌───────────────────────────────────────────────┐
│  标题：辐射换热 · 冷库屋面的太阳得热              │
├───────────────────────────────────────────────┤
│      ☀（可沿天空弧线拖动/自动昼夜循环）          │
│        ↘ 辐射箭头（粗细∝辐照度 I）               │
│  ▓▓▓▓▓ 屋面（可切白/浅灰/深/黑）▓▓▓▓▓  ← 表面温度色带  │
│  [库内 -18℃]                                    │
├───────────────────────────────────────────────┤
│  右侧仪表：α_s | I=800 | 综合温度 t_sa | q 漏热   │
│  T⁴ 曲线：横轴 T(K)，纵轴 σT⁴，标出当前工作点     │
├───────────────────────────────────────────────┤
│  控制：屋面颜色切换 · 时间滑杆(昼夜) · 天空晴/阴   │
└───────────────────────────────────────────────┘
```

**视觉语言**：一个真实的"天空—太阳—屋面"场景（呼应"辐射来自天上、无需介质穿越结构"），辐射用**从太阳射向屋面的箭头/光锥**表达，箭头粗细/亮度 ∝ 辐照度。屋面用色带显示因吸收而升高的表面温度（深色明显更红）。

---

## 4. 关键交互组件与实现逻辑

### 4.1 天空 + 太阳（场景层）
- **Canvas** 绘制渐变天空（时间→天色：清晨/正午/黄昏/夜晚），太阳沿半圆日轨运动。
- 太阳位置（高度角）决定投射到屋面的辐照度 $I = I_{max}\cdot\max(0,\sin(\text{高度角}))\cdot\text{晴空系数}$。
- 可拖动太阳，或点"播放"自动昼夜循环。

### 4.2 屋面颜色/反射率切换（核心开关）
- 一组材质按钮：白（α_s=0.25）、浅灰（0.5）、深（0.75）、黑（0.9）。
- 切换即改 α_s → 重算综合温度与漏热 → 屋面色带与仪表跳变。
- **对比模式**：可并排显示白/黑两块屋面同时受晒，一目了然差距（呼应例题的 12℃ vs 36℃）。

### 4.3 T⁴ 曲线（物理直觉层）
- **SVG** 画 $E=\varepsilon\sigma T^4$ 曲线，横轴 T(K)。
- 在曲线上打一个随屋面表面温度移动的"工作点"，让用户看到温度小幅上移时纵坐标陡增——把"四次方"具象化。
- 可叠加太阳(≈5800K，短波)与屋面(≈320K，长波)两点，直观区分短波/长波。

### 4.4 仪表与综合温度
- 实时显示 α_s、I、$t_{sa}=t_o+\alpha_s I/h_o$、漏热 $q=K(t_{sa}-t_i)$。
- 提供"若换成 KP01 的白墙 K 值"联动，把屋面得热接入围护结构漏热总账。

---

## 5. 页面实时计算模型

```
I = I_max * max(0, sin(sunAltitude)) * skyFactor      // 随日轨/天气
t_sa = t_o + α_s * I / h_o        // 太阳综合温度（可含长波修正项）
q = K * (t_sa - t_i)              // 与 KP01 的 K 衔接
E_surface = ε * σ * (T_surface_K)^4   // T⁴ 曲线工作点
```

均为闭式表达式，`requestAnimationFrame` 逐帧更新即可流畅呈现昼夜循环。

---

## 6. 可视化技术选型（为何这样选）

- **Canvas 画天空/太阳/光锥**：需要渐变、动画、逐帧昼夜过渡，Canvas 的像素级绘制与动画性能最合适。
- **SVG 画 T⁴ 曲线与工作点**：曲线需精确坐标、可标注、可交互悬停，SVG 更利落。
- 二者**分层叠放**（场景 Canvas 在下，数据 SVG/HTML 仪表在上），各取所长。这与 KP01 纯 SVG 剖面的选型不同——因为本页有"氛围动画"需求。

---

## 7. 与主"数字冷库"页面的衔接

- 入口：主场景点击**屋面/顶板**热点 → 浮窗"辐射换热……" → "深入了解"进入本页。
- 参数回带：本页选定的屋面颜色/反射率、当前太阳辐照，可作为屋面边界条件传回主站或 KP01 页，形成"屋面得热 → 顶板漏热"的连贯链路。
- 身份标识：**昼夜天空 + 太阳日轨**是本页独有形态，绝不与其它页面雷同。

---

## 8. 技术栈建议

- Canvas 2D（场景/动画）+ SVG/HTML（曲线与仪表）；可加轻量补间库做太阳平滑运动。
- 若主站 React：场景用 `<canvas>` + `useRef` 手绘，仪表用组件；或用 `react-konva` 管理场景层。
- 纯前端，无需后端。

---

## 9. 实现里程碑

1. 静态屋面 + 固定太阳，跑通 α_s → t_sa → q 计算与仪表。
2. 加屋面颜色切换，验证白/黑对比差异（对齐例题）。
3. 加太阳日轨与昼夜循环动画，I 随高度角变化。
4. 加 T⁴ 曲线与工作点联动。
5. 加白/黑并排对比模式与天气（晴/阴）系数。

---

## 10. 验收标准

- 切换屋面颜色，太阳当量温升与漏热按 α_s 成比例跳变（白≈12℃、黑≈36℃对齐例题）。
- 拖动/播放太阳，辐照度、综合温度、漏热随昼夜起落，夜间趋近纯对流。
- T⁴ 曲线让"温度稍升辐射剧增"可被肉眼感知。
- 页面形态（**天空+太阳+屋面颜色**）**只可能属于"辐射换热"**——达成"不套模板、精准契合"。
