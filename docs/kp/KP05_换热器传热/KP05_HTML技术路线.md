# KP05 · HTML 技术路线：换热器传热

> 说明"换热器传热（蒸发器）"专属 HTML 页面**如何设计**，非代码交付。

---

## 1. 为什么这个知识点适合这样呈现？

换热器最经典、最不可替代的可视化是**"温度—流程(长度)图"**：横轴是沿盘管流程的位置，纵轴是温度，画出**空气侧下降曲线**与**制冷剂侧恒温平线**两条线，它们之间夹着的就是**沿程温差**，而 **LMTD 就是这块温差的"对数平均高度"**。这张图是别的 10 个知识点都不会有的，天生属于换热器：
1. **两种流体、沿程变化** → 双温度曲线 vs 流程。
2. **制冷剂相变恒温** → 一条水平线（辨识度极高）。
3. **LMTD/端差** → 两曲线间可填充的温差带 + 端差 ΔT₁、ΔT₂ 标注。
4. **U 由串联热阻合成** → 一条"空气→翅片→管壁→制冷剂"的**热阻串联条**。

因此本页核心是**"沿程温度双曲线图 + LMTD 温差带 + 热阻串联条 + U·A·ΔT 面板"**——一个**"过程曲线 + 性能计算器"**页面。

---

## 2. 核心交互隐喻

**"拖动风量/蒸发温度，看空气曲线和制冷剂平线之间的温差带如何胀缩，LMTD 和换热量随之变。"** 这是本页独有记忆点。

## 4. 关键交互组件与实现逻辑

### 4.1 温度—流程双曲线图（主视图，本页灵魂）
- **SVG/Canvas** 折线：空气温度沿流程按指数逼近制冷剂温度下降；制冷剂为水平线。
- 两线之间**填充半透明温差带**；端点标 ΔT₁、ΔT₂；在带中画一条虚线代表 LMTD 高度。
- 拖动参数时两曲线与温差带实时重绘。

### 4.2 热阻串联条（把 U 拆开看）
- 一条水平条分三段：空气侧(含翅片)、管壁、制冷剂侧，**每段宽度 ∝ 其热阻**。
- 直观显示"空气侧最宽（热阻最大，瓶颈）"，呼应"为何加翅片"。悬停高亮与主图/面板联动。

### 4.3 性能面板（LMTD 与 ε-NTU 双算法）
- 同时展示 LMTD 法（$Q=UA\,\Delta T_{lm}$）与 ε-NTU 法（$\varepsilon=1-e^{-NTU}$）结果，二者应一致，互为印证。
- 显示 ΔT₁、ΔT₂、LMTD、U、A、Q、NTU、ε。

### 4.4 控制与联动
- 风量滑杆→空气侧 $h_o$（接 KP06 思想）；蒸发温度、库温、翅片面积可调。
- "叠加霜层"开关：加一层污垢/霜热阻（接 KP07），观察 U 下降、Q 缩水、温差带变化。

## 5. 页面实时计算模型

```
// 空气侧热容流率
C_air = m_dot_air * cp_air
UA = 1 / ( 1/(η_o*h_o*A_o) + ln(ro/ri)/(2πλL) + 1/(h_i*A_i) + R_frost )
NTU = UA / C_air
ε   = 1 - exp(-NTU)                        // 一侧相变
Q   = ε * C_air * (T_air_in - T_evap)
T_air_out = T_air_in - Q / C_air
ΔT1 = T_air_in - T_evap;  ΔT2 = T_air_out - T_evap
LMTD = (ΔT1 - ΔT2) / ln(ΔT1/ΔT2)           // 与 Q=UA*LMTD 应自洽
// 空气沿程曲线： T_air(z) = T_evap + (T_air_in - T_evap)*exp(-UA*z/L /C_air)
```

两种方法自洽是很好的"自检"，也是教学亮点。

## 6. 可视化技术选型（为何这样选）

- **SVG（温度-流程图 + 热阻条）**：需要精确坐标轴、可标注端差、可填充温差带、可悬停，SVG 最合适；曲线点不多，无性能压力。
- 若想让空气曲线"流动"（沿程动画箭头），可叠一层 Canvas 画流向粒子，但非必需。
- 与前面粒子流/温度场页面区别：本页是**函数曲线 + 工程计算器**风格——因为换热器的本质就是"沿程过程 + 集总性能"。

## 7. 与主"数字冷库"页面的衔接

- 入口：主场景点击**蒸发器盘管**热点 → 浮窗"换热器传热……" → "深入了解"进入本页。
- 联动：本页是多个知识点的"汇流处"——空气侧 h_o 来自 KP06、管壁来自 KP11、霜层来自 KP07、相变来自制冷剂潜热。可在页内放"跳转到该子知识点"的锚点，形成教学网络。
- 身份标识：**温度-流程双曲线 + 恒温平线 + LMTD 温差带**是本页独有形态。

## 8. 技术栈建议

- SVG 曲线 +（可选）Canvas 流向层；面板用 HTML/组件。
- 状态 `{m_dot_air, T_air_in, T_evap, A, fins, frost}` → 派生全部性能量。
- 纯前端，无后端。

---

## 9. 实现里程碑

1. 画静态温度-流程图（写死参数），标端差与 LMTD。
2. 接 LMTD 与 ε-NTU 双算法计算并互校（对齐例题 LMTD≈7.83、UA≈1277）。
3. 加控制滑杆，曲线/温差带/面板联动。
4. 加热阻串联条与悬停高亮。
5. 加霜层开关，观察 U/Q 变化（接 KP07）。

---

## 10. 验收标准

- 拖动参数，空气曲线与温差带实时变化，LMTD 法与 ε-NTU 法结果一致。
- 数值与 KP05 例题一致（LMTD≈7.83 ℃、UA≈1277 W/K）。
- 热阻条清晰显示"空气侧最宽（瓶颈）"，叠加霜层后 U/Q 明显下降。
- 页面形态（**沿程温度双曲线 + 恒温相变平线**）**只可能属于"换热器"**——达成精准契合。
