# KP06 · HTML 技术路线：强制对流

> 说明"强制对流（蒸发器风机）"专属 HTML 页面**如何设计**，非代码交付。

---

## 1. 为什么这个知识点适合这样呈现？

强制对流与自然对流（KP04）都是"流动"，但**驱动源不同**——这里有一台**风机**在主动吹。页面必须让"风机是主控旋钮、流速是自变量、h 是因变量"这条因果链成为主角，从而与 KP04 的"温差自发流动"形成鲜明对照：
1. **风机驱动** → 页面有一个显眼的**风机转速旋钮/滑杆**，是一切的入口。
2. **空气外掠管束** → 主视觉是**空气流线/粒子横掠一排盘管**，流速快时流线密集、边界层被吹薄。
3. **Re→Nu→h 幂律** → 天然对应一条 **h∼v^0.6 的增益曲线**与实时 $h$ 表盘。
4. **副作用（能耗³/干耗/内热）** → 天然对应"收益 vs 代价"的权衡仪表。

因此本页核心是**"风机转速旋钮 → 掠管流线场 → 边界层变薄 → Re/Nu/h 表盘 + 能耗/干耗代价"**，是一个**"旋钮驱动的因果链"**页面。

---

## 2. 核心交互隐喻

**"拧大风机转速，看流线掠过盘管、贴壁边界层被吹薄、h 表盘蹿升——但旁边的风机能耗按立方猛涨。"** 这是本页独有记忆点：**一个旋钮，两条曲线（收益 vs 代价）。**

## 3. 页面布局与视觉结构

```
┌───────────────────────────────────────────────┐
│  标题：强制对流 · 蒸发器风机                      │
├───────────────────────┬───────────────────────┤
│  🌀风机→ ⇒⇒⇒ 流线掠过   │  Re | Nu | h 表盘       │
│         ◯ ◯ ◯ 盘管排    │  h∼v^0.6 增益曲线(工作点)│
│         ▏边界层(随v变薄) │  Gr/Re² 判据: 强制/自然  │
│         ⇒⇒⇒            │                        │
├───────────────────────┴───────────────────────┤
│  代价仪表: 风机功率∼v³ | 内热负荷 | 干耗速率       │
│  控制: 风机转速/风速 v · 管径 D · 空气温度         │

```

**视觉语言**：左侧风机 → 一束**从左向右的流线/粒子**掠过一排圆形盘管截面；每根管背风侧可画尾流；管壁贴一层**随 v 变薄的边界层**。右侧是把 v 映射成 h 的曲线与表盘。整体传达"机械驱动、单向吹送"。

---

## 4. 关键交互组件与实现逻辑

### 4.1 掠管流线场（主视图，本页灵魂）
- **Canvas 粒子/流线**：粒子从风机侧持续注入，横向掠过管束，遇管绕流（可用简化绕流位移或预设流线纹理）。
- 风速 v 提高 → 粒子速度、流线密度上升；管壁边界层薄膜厚度 ∝ $1/\sqrt{Re}$ 变薄。
- 与 KP04 的**竖直浮升粒子**明确区别：这里是**水平强制吹送**。

### 4.2 Re→Nu→h 曲线与表盘
- **SVG** 画 $h(v)\propto v^{0.6}$ 曲线，工作点随旋钮移动；表盘显示 Re、Nu、h。
- 标注层流/湍流转捩，呼应 $Nu=C\,Re^mPr^n$ 分段。

### 4.3 收益 vs 代价仪表（本页教学升华）
- 三个代价读数：风机功率（∝v³）、风机内热负荷、货物干耗速率（∝ 换热/流速的经验式）。
- 可画"净收益"曲线（换热收益 − 能耗代价），标出最优风速——让"越大越好"的直觉被数据纠偏。

### 4.4 主导机制判据（承接 KP04）
- 显示 $Gr/Re^2$：风机关时→自然对流主导（跳转/联动 KP04），风机开时→强制对流主导。一个开关把两页物理串起来。

## 5. 页面实时计算模型

```
Re = v * D / ν
Nu = C * Re^m * Pr^n          // 外掠管束关联式，分段取 C,m
h  = Nu * λ / D
δ_bl ∝ D / sqrt(Re)           // 边界层厚度示意
P_fan ∝ v^3                    // 风机功率随风速立方
dryLoss ∝ h * (p_sat_surface - p_air)   // 干耗速率(传质，示意)
GrRe2 = Gr / Re^2             // 主导机制判据
netBenefit = a*h - b*P_fan    // 净收益(教学用，系数可调)
```

`h∼v^0.6`、`P∼v^3` 的强烈反差是本页要传达的核心。

---

## 6. 可视化技术选型（为何这样选）

- **Canvas/WebGL 粒子（掠管流场）**：强制对流的主角是"有向流动 + 绕流 + 边界层"，需要连续粒子动画，Canvas/`pixi.js` 合适。
- **SVG（h-v 曲线、代价曲线、表盘）**：函数曲线与工作点，SVG 清晰可交互。
- 与 KP04 同为粒子系统但**流向、驱动、判据**都不同——技术同类、语义相反，正好凸显两种对流的区别。

---

## 7. 与主"数字冷库"页面的衔接

- 入口：主场景点击**蒸发器风机**热点 → 浮窗"强制对流……" → "深入了解"进入本页。
- 联动：算出的空气侧 $h_o$ 回传 KP05（提高 U）与 KP08（货物表面换热）；风机内热回传库房热负荷总账；$Gr/Re^2$ 开关联动 KP04。
- 身份标识：**风机 + 水平掠管流线 + h∼v^0.6/能耗∼v³ 双曲线**是本页独有形态。

---

## 8. 技术栈建议

- Canvas 2D 或 `pixi.js` 粒子 + SVG 曲线/表盘。
- 状态 `{v, D, T_air}` → 派生 Re/Nu/h、边界层、能耗、干耗、判据。
- 纯前端，无后端。

---

## 9. 实现里程碑

1. 静态管束 + Re/Nu/h 计算与表盘（对齐例题 h≈53.3）。
2. 加掠管粒子流，速度/边界层随 v 变化。
3. 加 h-v 增益曲线与工作点。
4. 加能耗∼v³、干耗、净收益曲线与最优点。
5. 加 Gr/Re² 判据开关，联动 KP04/KP05。

---

## 10. 验收标准

- 拧大风机，流线加密、边界层变薄、h 按 v^0.6 上升；同时能耗按 v³ 猛涨、净收益出现峰值。
- 数值与 KP06 例题一致（3 m/s 时 h≈53.3；翻倍约 ×1.52）。
- 风机关闭时判据切到自然对流主导，可联动 KP04。
- 页面形态（**风机水平掠管 + 收益/代价双曲线**）**只可能属于"强制对流"**——与 KP04 精准区分。
